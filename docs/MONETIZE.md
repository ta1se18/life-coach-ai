# MONETIZE

## 目的
この文書は、個人収益としての「月10万マネタイズ」を軸に、実行可能な収益化戦略と評価指標を整理する。

## 目的
- 個人のキャッシュフローを安定させる
- 事業運営のリスク分散を図る
- 収益化の学びを事業OSに還流する

## 現在の目標
- **短期目標：** 3〜6ヶ月以内に月10万円を実現する
- **中期目標：** 1年以内に安定的な収益基盤を構築する
- **長期目標：** 個人活動としての収益源を拡張し、共同事業のリソースを支える

## 収益チャネル候補
### 1. コンテンツ・情報販売
- 小規模なノウハウ教材、テンプレート、分析レポート
- AI活用や営業対話の学びをコンテンツ化

### 2. コンサル / 伴走サービス
- AO塾向け営業改善支援
- 対話構造設計のコンサルティング
- すぐに検証できる小規模案件を中心に受注

### 3. デジタルプロダクト
- LIFE COACH AIの付加価値機能やテンプレート
- Web版を活かしたプレミアム機能の検討

### 4. SNS / Community-driven
- 情報発信からのリード獲得
- 小規模有料コミュニティやメルマガ

## KPI
- 月間収益額（目標：10万円）
- 収益源の数と比率
- 収益化施策ごとの粗利
- リード獲得数とCV率
- コンテンツ/サービスの再現性

## 戦略
### 1. まずは「最も実行しやすい収益源」に集中
- 小さく始めて、早期に収益化を確認する
- 導入価格帯は低めに設定し、反応を見ながら拡大する

### 2. 事業OSの知見を活かす
- 「対話構造」「意思決定の変化」に関する知見を商品化する
- 営業改善やAO塾支援を、個人マネタイズのリードにする

### 3. マネタイズは個人の専用戦線として運用
- 共同事業の学びを活かすが、収益は個人の責任範囲とする
- 事業OSとは目的を切り分け、混同しない

## 運用ルール
- 収益施策は週次レビューで見直しを行う
- 施策ごとに仮説と検証指標を定義する
- 収益が出なければ、すぐに改善案か別施策に切り替える
- まずは1〜2本の収益チャネルに集中し、拡大は安定後に行う
