# STARTUPS

## 目的
この文書は、ミズキと進める複数のスモール起業を整理し、共同事業と個人活動を分離しながら実行するための運用ドキュメントである。

## 構造
- **共同：** ミズキとのスモール起業群
- **目的：** 小規模で実験的に事業を回し、共同事業の学びと資金を作る
- **位置づけ：** 営業OS本命とは別軸のスモールビジネスポートフォリオ

## 方針
- まずは短期間で検証できるビジネスを複数用意する
- 事業の規模は小さめにし、素早い仮説検証を優先する
- ミズキとの共同運営体制を明確にし、責任範囲を分担する
- スモール起業群は共同事業のキャッシュ創出かつ実行力強化の場とする

## 具体例
- **教材・学習支援サービス**
- **小規模コンサルティング / 伴走サービス**
- **コミュニティ運営型の情報提供**
- **シンプルなデジタルプロダクト**

## 運用原則
1. **実行のしやすさを最優先**
   - 低コスト・短期間で立ち上げられるサービスを選ぶ
2. **明確な検証指標を設定**
   - 収益、顧客獲得、反応、改善サイクル
3. **学びを共同事業へ還流**
   - 対話設計、顧客理解、リード獲得プロセスの知見を共有
4. **個人収益と混同しない**
   - 共同起業の収益は共同事業として扱い、個人マネタイズは別にする

## KPI
- 各スモール事業の月次収益
- 検証サイクルの速度
- 顧客フィードバックの質
- 共同作業の実行効率
- 共同でつくるノウハウ蓄積量

## 運用ルール
- 週次で各スモール起業の進捗を振り返る
- 成果が出ない事業は早めに撤退またはピボットする
- 事業OSとの連携機会を常に探し、可能なら共同知見として蓄積する
- 共同事業と個人活動のリソース配分を明確に記録する
