# APP

## 目的
この文書は、LIFE COACH AIのアプリ開発構想と現状を整理し、個人活動としての開発戦略を明確にするためのドキュメントである。

## アプリの位置づけ
- **目的：** 人生管理・自己理解・行動支援を行うAIアプリを実現し、将来的にAI実行OSと接続する。
- **現時点の役割：** 個人活動のサポートと、将来のサービス化に向けたMVP検証。
- **方針：** まずは「確実に動くこと」を優先し、Web版の安定運用を重視する。

## 現状
- GitHub Pagesで公開済み
- フォームUIは完成に近い
- 年齢入力、今日の状態、自己理解ヒアリング、深掘り質問、人生残り時間表示などが動いている
- 現在の課題は主にWeb版のUI安定性

## 現在の課題
- Expo Webで一部の `TouchableOpacity` がスマホWeb上で反応しない
- 特に「AIの話し方ボタン」「性別ボタン」のタッチ反応が不安定
- WebとモバイルWebでの安定性を確保する必要がある

## 優先対応
1. **Webで安定動作するフォーム設計**
   - `TouchableOpacity` 依存のボタンを一旦 `TextInput` や標準フォームコントロールに切り替える
   - フォーム入力のアクセシビリティを確保する
2. **remainingYearsを入力即時反映**
   - 年齢入力値からの残り時間表示をリアルタイムに更新する仕様
3. **AI分析ボタンを中核機能として維持**
   - AI分析機能の導線を明確にし、動作を優先する
4. **既存デザインはできるだけ保持**
   - 既存のUI設計を尊重しつつ、安定動作を優先
5. **モバイル最適化**
   - スマホWebでの入力しやすさ、操作性、表示崩れを重点的に確認
6. **GitHub Pages前提で壊れない構成**
   - 静的サイトとして公開できる状態を維持する

## 技術スタック
- Expo
- React Native Web
- TypeScript
- GitHub Pages

## 開発方針
- **まずは運用に耐えるMVPを優先**し、機能拡張やデザイン改善は後回しにする。
- **動くことを最優先に**、安定した入力体験とAI分析の可用性を確保する。
- **事業OSとの接続は将来的目標**として維持し、現在は個人活動の開発として進める。

## 次のアクション
- Webフォームの不具合箇所を特定し、安定化対応案を作る
- 今日の段階では、UIをシンプル化して動作保証を優先する
- 将来的に、アプリは共同事業のデータ蓄積と意思決定支援に接続する予定である
