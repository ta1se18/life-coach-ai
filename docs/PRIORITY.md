# PRIORITY

## 目的
この文書は、現在の優先順位とそのルールを明確にし、共同事業と個人活動の意思決定を一貫して運用するための基準を示す。

## 現在の優先順位（全体）
1. 事業（営業OS / AO塾の再現性証明）
2. 月10万マネタイズ
3. アプリ開発
4. TOEIC
5. 読書

## カテゴリ別の優先順位
### 共同事業
- **営業OS（本命長期事業）**
- **みずきと行う複数スモール起業**

### 個人活動
- **月10万マネタイズ**
- **アプリ開発**
- **TOEIC**
- **読書**

## 優先順位の考え方
- 共同事業は長期的な価値生成と再現性証明を最優先とする。
- 個人活動は、共同事業を支援するリソースと自己成長を両立させるために優先度を設定する。
- 事業と個人は切り分けるが、相互に学びを還流させる。

## 各項目の目的と時間軸
### 事業
- 目的：AO塾で意思決定構造の再現性を証明し、営業OSの基盤を作る。
- 時間軸：今〜1年で再現性検証、1〜3年でAI支援サービス化。
- KPI：AO塾1社での対話データ取得数、意思決定変化ポイント抽出率、検証プロジェクト進捗。

### 月10万マネタイズ
- 目的：個人の収益基盤をつくり、事業運営の現金流を補完する。
- 時間軸：今〜半年で月10万円、半年〜1年で安定化。
- KPI：月間収益額、収益源の種類、再現性のある収益モデル数。

### アプリ開発
- 目的：AI実行組織の概念を支えるツールを育てる。現段階では運用支援として構築。
- 時間軸：今はMVP検証段階、運用体制確立後に本格展開。
- KPI：Web版の安定動作率、フォーム完了率、AI分析ボタン起動率。

### TOEIC
- 目的：英語力とグローバル思考のベースを作る。
- 時間軸：10月までに700点を目標。今後6カ月の主要学習目標。
- KPI：模試スコア、学習時間、単語習得数。

### 読書
- 目的：抽象化能力・事業理解・思考力の強化。
- 時間軸：継続的習慣化。月ごとの読了数と知識定着。
- KPI：読了冊数、アウトプット数（メモ・記事・会話への反映）。

## 優先順位変更ルール
1. **事業機会が明確に拡大したとき**は、事業優先をさらに強化する。
2. **収益化の緊急性が高まったとき**は、月10万マネタイズを優先度2へ維持する。
3. **アプリ開発の実運用価値が明確になったとき**は、アプリ開発の優先度を上げる。
4. **TOEIC試験や読書の期限が近いとき**は、個人学習の時間配分を調整する。

## 運用ルール
- 毎週の週次レビューで優先順位を確認し、必要な修正をまとめる。
- 大きな方針変更は、あなたの最終承認を得てから反映する。
- 日々の工程表は、この優先順位を基準に調整する。
