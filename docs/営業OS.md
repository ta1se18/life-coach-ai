# 営業OS

## 目的
この文書は、共同事業の中核である「営業OS / 意思決定構造化事業」の概念と運用方針を整理する。

## 事業の本質
- 営業や対話から「結果」ではなく「原因（なぜ）」を取得し、構造化する。
- 営業は「売る行為」ではなく「欲求を発見する行為」である。
- 対話の中で顧客の不安、価値観、潜在ニーズ、感情変化を抽出する。

## フェーズ
### フェーズ1：コンサル型
- 現在のフェーズ
- 人間がフロントに立ち、現場理解とログ取得を優先する。
- 営業改善・面談改善・対話設計を伴走する。

### フェーズ2：半自動化
- 会話分類、感情変化検知、仮説生成をAIで補助する。
- ここでAIの価値が明確になる。

### フェーズ3：SaaS化
- 十分なデータと再現性が蓄積された後に、営業AI / 意思決定分析OSとして展開する。

## 初期ターゲット
- **AO塾**：高関与意思決定・対話量が多く、変数が比較的少ない市場
- 代表候補：ルークス、早稲田塾、ナガセ

## AI活用の焦点
- **データ設計が重要**であり、AIは手段に過ぎない。
- 重点項目：
  - テキスト分類（不安・比較・興味・拒否・納得）
  - 重要発言の抽出
  - 感情変化の検知
  - プロンプト設計

## 競合との違い
- 既存競合：Gong、Chorus、Salesforce、HubSpot、People.ai、Clari
- 既存：何が起きたかを記録
- 本事業：なぜそうなったかを理解
- 差別化点：意思決定理解が中心

## 最重要事項
- 世界観拡大よりも、**AO塾1社での再現性証明**を最優先する。
- 取得すべきデータ：
  - どこで意思決定が変化したか
  - どの対話が効いたか
  - 何が感情変化を起こしたか

## 運用ルール
- 共同事業は検証サイクルを高速に回す。
- データと検証結果を基に仮説を更新する。
- 共同事業と個人活動は目的とリソースを分離し、混同しない。
- 週次レビューで進捗と学びを必ず確認する。
