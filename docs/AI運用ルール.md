# AI運用ルール

## 目的
この文書は、Claude Codeを中心としたAI実行組織の運用ルールを具体化し、運用の一貫性と品質を担保する。

## 役割分担
### あなた（人間）
- 最終意思決定者
- 戦略と優先順位の決定
- 重要なビジネス判断
- 品質承認
- 倫理・法務・リスク判断

### Claude Code
- AI執行ディレクター
- 毎朝の工程表生成
- タスク管理と進捗支援
- 情報収集・分析・提案作成
- 他AIの使い分けと品質管理
- 改善提案の立案

## 指示の出し方
### 基本
- 毎朝は「おはよう」で工程表を生成する。
- 具体的な指示は「目的」「期限」「優先度」「制約」を含める。

### フォーマット
1. 目的：何を達成したいのか
2. 期限：いつまでに必要か
3. 優先度：最優先 / 可能なら / 継続
4. 制約：時間やリソース、前提条件

### 例
- 「今週中にAO塾の対話ログ取得方針をまとめてほしい」
- 「今朝は月10万マネタイズの収益案を2つ提案してほしい」

## 社内AIの使い分け
### Claude Code
- 戦略立案・統合提案
- 複雑な思考整理・品質レビュー
- 運用の司令塔

### ChatGPT-4
- 文章作成（提案書・営業文面・SNS投稿）
- アイデアブレインストーミング
- コンテンツ生成

### GitHub Copilot
- コード生成・実装支援
- バグ修正補助
- 技術調査

### Perplexity等のリサーチAI
- 市場調査
- 競合リサーチ
- 最新トレンド把握

## 判断分担
### 人間が判断すべきこと
- 優先順位の変更
- 戦略の方向性
- 収益化に関する重大判断
- 重要な契約や協業の判断
- 予期せぬリスクへの対応

### Claude Codeが自律的に進めるべきこと
- 日々のスケジュール生成と調整
- タスクの実行支援と進捗管理
- 情報収集とレポート作成
- 他AIの活用と出力品質チェック
- ルールに沿った改善提案

## 運用ルール
- 日次：毎朝「おはよう」で計画生成、毎夕に日次振り返り
- 週次：週次レビューで優先順位・施策を調整
- 月次：月次レビューで目標・戦略を再確認
- ルール変更は記録し、次回以降に反映する

## 長期成長のためのルール
- AIの結果は常にあなたが最終確認する
- 運用履歴を記録し、次回以降の改善材料とする
- AIが独自に拡大する際は、事前に範囲と条件を明確に定義する
- 日本語で思考を維持し、言語の解像度を担保する
